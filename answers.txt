Q1. What is a gitignore file and what should it include for a Java project? 
A1. Gitignore file is text file which indicate git to ignore specific files and folders in project. In java project gitignore should include
*.class, *.log, *.ctxt, .mtj.tmp/, *.jar, *.war, *.nar, *.ear, *.zip, *.tar.gz, *.rar, hs_err_pid* .

Q2. What does fast-forward in git mean? 
A2. when git move forward the source branch pointer to the target branch pointer without even making an new merge commit.

Q3. Why should you have different branches? 
A3. Having branches in project can help a lot like managing multiple branches of the assigned task can make their own changes without affecting others work and merge in main/master branch in the end for collobrative work.

Q4. In a project how many remote Git repositories do you usually have? 
A4. We usually have one repository per project however in big project like commercial web-applications having multiple repositories of modules help a lot for managing the project.
